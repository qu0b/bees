# Bees — Autonomous Multi-Agent Orchestration Platform

A CLI/daemon that autonomously builds and evolves software projects. `bees init` analyzes a codebase, generates configuration, and `bees daemon` runs continuously — spawning AI agents that implement features, review code, merge changes, run tests, deploy, and decide what to build next. The human provides a project and target users; bees does the rest.

The system accumulates knowledge as it works. Not documentation (which rots), but a persistent knowledge base of ground truths: architecture decisions, component relationships, failed approaches, design rationale. Agents read this knowledge before acting and write back what they learn. Over time, the swarm develops deep institutional memory about the project it's building.

## Tech Stack

- **Zig 0.16.x** preview (async I/O via io_uring green threads). Installed at `/opt/zig`.
- **LMDB** — vendored C, zero-copy mmap reads, bit-packed records
- **SQLite** — vendored amalgamation, WAL mode, queryable read replica synced from LMDB
- **DuckDB** — runtime `dlopen`, ATTACHes SQLite directly for analytical queries
- **Multi-backend** — Claude CLI (primary), OpenAI Codex, OpenCode, Pi. Configurable per-role.
- Schema defined once in `src/db/schema.zig` (comptime), generates DDL + bind functions for all databases

## Architecture

**Autonomy model** — `bees init` analyzes a codebase and bootstraps the full configuration: roles, prompts, workflows, user profiles. `bees daemon` runs a continuous cycle defined entirely by the workflow — which roles run, in what order, under what conditions. The workflow is the brain; the executor is the muscle. No human in the loop unless they choose to steer.

**Roles** are user-defined in `.bees/roles/{name}/config.json + prompt.md`. Model, effort, budget, security profile, MCP plugins — all configurable per role. The system ships with sensible defaults but everything is overridable.

**Executor** (`src/executor.zig`) — generic role runner. Standard lifecycle: create session → build prompt with context → run agent → update status → store report. Workers and merger have specialized paths (`src/worker.zig`, `src/merger.zig`) due to worktree management and merge logic; all other roles go through executor.

**Workflow** (`src/workflow.zig`) — declarative JSON sequence of steps. Periodic (`every: N`), conditional (`condition: "tool_errors"`), parallel groups. Defines the entire orchestration cycle.

**Task execution** — worker roles spawn in isolated git worktrees, pick tasks from a weighted pool, implement and commit. Restart-on-timeout with session resume. A merge role scans completed worktrees, reviews diffs, merges, and runs the build/test/deploy pipeline with AI-assisted fix on failure.

**Knowledge base** (`src/knowledge.zig`) — `.bees/knowledge/` — the swarm's institutional memory. Markdown files organized by category (architecture, components, contracts, decisions, failed, operations) with a compact JSON index in LMDB (`kb:index` meta key). Agents write knowledge by including `## Knowledge Updates` sections in their output — the executor parses CREATE/UPDATE/APPEND directives, writes files, updates the index, and appends to `_log.md`. Agents read knowledge via tag-filtered context injection with a configurable budget cap. The `_schema.md` conventions document (generated by `bees init`) governs structure. Any role can be configured to read/write knowledge — it's just another context source.

**Researcher** — autonomous code analysis agent that builds the knowledge base. Reads existing knowledge to find gaps, investigates deeply (code, git history, tests, web search), and writes structured knowledge updates. Runs every 2 merge cycles by default, before the strategist so it has fresh knowledge for planning. Uses Opus model for deep reasoning. Security profile: read-only code + web search + test running.

**Context assembly** (`src/context.zig`) — each role declares which sources it needs (user profiles, operator feedback, reports, task trends, worker summary, changed files, knowledge base). The context module loads them in a single LMDB read transaction and injects into the agent prompt. Knowledge sources use `"knowledge:<tag>"` or `"knowledge:*"` syntax in role configs.

**Config** at `<project>/.bees/config.json` + `tasks.json`, auto-detected by walking up from CWD. REST API (port 3002) for dashboard. Security profiles per-role via `--allowedTools`/`--disallowedTools`.

## Data Model

**Packed structs** with comptime size assertions:
- `SessionHeader`: 48B. `EventHeader`: 4B. `ReviewHeader`: 8B. `TaskHeader`: 16B.
- Enums use minimum bit widths: SessionType(u4), SessionStatus(u3), EventType(u3), ToolName(u4).
- Monetary values as integer microdollars. Timestamps as u40 unix seconds. Never f64, never ISO strings.
- Sentinels over optionals — avoids tag byte + alignment padding.

**LMDB** — 7 sub-databases. Big-endian keys. `MDB_RESERVE` for zero-copy writes. `MDB_NOTLS` for green threads. Directory mode only. Event writes are non-fatal — dead letter queue for failed writes.

**Zero-copy everywhere** — slices into owned buffers, never duplicate unless crossing ownership boundaries. NDJSON parser reuses a single line buffer.

**Allocator scopes** — global arena (config, prompts), per-session (branches, worktrees), stack buffers (formatting), LMDB read txn (mmap pointers). Target < 512 KB steady-state.

## Zig Coding Standards

**Safety > Performance > Developer Experience.** Zero technical debt — do it right the first time.

- `const assert = std.debug.assert;` at module scope. `std` imports first, then local modules.
- **Assert preconditions, postconditions, and state transitions.** Split compound assertions. Pair assertions at write AND read paths.
- **Comptime assertions** for struct sizes, bit widths, and cross-struct relationships.
- **`assert`** = programmer bug. **`unreachable`** = logically impossible. **`@panic`** = impossible runtime condition with message.
- **`errdefer`** immediately after each fallible operation. `switch` on errors, never `== error.`.
- **≤70 lines per function.** Decompose into focused helpers. Public functions at top of file.
- **Exhaustive switches** on enums — list every variant. `else =>` hides new variants from the compiler.
- **All loops bounded.** Safety counters on dispatch loops. No unbounded recursion.
- **`@divExact`/`@divFloor`** to show division intent.
- **Units last** in names: `timeout_secs`, `cost_microdollars`. No abbreviations.
- **`zig fmt` always.** Trailing commas. 100-column limit. `defer` followed by blank line.
- **One generic solution** over N copies. No role-specific modules — everything through executor.
- Comments are sentences. Always say *why*.

## Key Constraints

- Deploy capability is restricted to the merge phase — task execution roles must never deploy
- Model, effort, budget, security profile all configurable per-role
- Build/test/deploy commands configurable per-project in config.json
- Worktrees managed by bees for full control over naming, state, cleanup
- The workflow definition is the single source of truth for what runs, when, and under what conditions
- Reports derived on demand from LMDB meta — not written to disk as files
- Knowledge base is append-mostly — agents add and refine, never bulk-delete
